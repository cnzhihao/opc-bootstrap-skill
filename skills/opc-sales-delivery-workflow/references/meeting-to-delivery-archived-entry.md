# OPC 会议纪要转交付动作

## 使用场景
当用户需要把一次客户会议、交付会议或项目沟通转成项目更新、行动项和交付节奏时使用。

典型触发：
- “把这次会议整理成交付动作”
- “这段会议纪要更新到项目里”
- “客户会后我们下一步该做什么”
- “帮我把会议结论同步到项目总表”
- “从这次沟通里提炼客户反馈和交付物变化”

## 核心定位
本 skill 是交付技能组的会议转行动工具，负责把单次会议的信息转成结构化交付更新。它不负责项目全局节奏判断，也不负责周复盘或案例完整沉淀。

P1 阶段只读取复用现有飞书文档和多维表，不主动新增字段。本 skill 可作为 `opc-project-rhythm-manager` 的前置解析器：当用户输入会议纪要、录音转写或客户沟通摘要时，先由本 skill 提取事实和行动项，再交给项目节奏 skill 做二次判断。

## 输入信息
优先使用：
- 会议纪要、录音转写、用户口述或客户沟通摘要。
- 项目名称或客户名称。
- 会议时间、参与人、背景。
- 相关交付物、项目文档或飞书链接。
- 可选：项目总表记录、今日作战台记录。

信息不完整时，责任人、截止时间、项目状态等应标记“待确认”，不要自行编造。

## 读取飞书落点
根据需要只读读取：
- `00｜OPC CEO 作战系统 / 项目总表`
- `00｜OPC CEO 作战系统 / 今日作战台`
- `03｜交付项目 / 对应项目文件夹或项目文档`
- `00｜OPC CEO 作战系统 / 客户线索`，仅当会议产生续费、转介绍或新需求线索时读取
- `02｜周月复盘库`，仅当会议产生复盘判断时参考

## 拟写入飞书落点
所有写入必须先展示拟操作并获得用户确认。

可拟写入：
- 脑内清空池：会议待办、action item、客户反馈、项目变化、经营判断、客户需求或内容选题必须先拟进入脑内清空池。
- 项目总表：仅当会议产生明确项目阶段、卡点、下一步动作、风险或客户反馈变化时，整理为可交给 `opc-feishu-update-review` 的拟更新项，经确认后再写入。
- 交付项目文档：追加会议纪要和行动项。
- 客户线索：仅当会议产生明确新商机时拟更新；仍需先在脑内清空池中完成经营判断。
- 内容工厂选题池：若会议中出现典型客户问题、异议、可公开表达的案例素材或内容选题，应先进入脑内清空池，再拟桥接给 `opc-content-factory-topic-picker` / `opc-content-factory-topic-prioritization`。

禁止本 skill 直接把会议待办或 action item 写入今日作战台。今日作战台只能由后续链路生成：脑内清空池 → OPC 节奏/复盘类 skill 判断并更新本周主战役 → `opc-daily-start-workflow` 基于本周主战役进入今日作战台。

## 工作流程
1. 解析会议内容。
2. 提取会议结论、客户反馈、行动项、责任人和时间节点。
3. 识别项目状态变化、交付物变化和风险变化。
4. 将会议待办、action item、客户反馈、项目变化和经营判断整理为拟进入「脑内清空池」的结构化条目，不直接写入今日作战台。
5. 识别是否产生续费、转介绍、案例或内容素材。
6. 生成会议转交付摘要。
7. 判断是否需要二次交给 `opc-project-rhythm-manager` 做项目阶段、卡点、风险和下一次关键交付动作判断；项目动作是否进入今日作战台由后续主战役和 daily-start 链路决定。
8. 生成拟写入飞书方案，并整理成可交给 `opc-feishu-update-review` 的结构化更新清单，等待确认。

## 今日作战台下游约束

本 skill 不直接把会议待办/action item 写入今日作战台。若后续经脑内清空池、主战役判断和 `opc-daily-start-workflow` 进入今日作战台，必须遵循：

- 每日作战台最多 5 件。
- 新增行动项默认「今日待开战」。
- 真正闭环用「已完成」；阶段性推进用「推进中」；等待外部反馈用「等外部反馈」。
- 延期承接拆成两条：原记录「已延期」，新承接记录「今日待开战」。
- 承接动作可更具体，但必须继承或更新「所属主战役」。
- 延期 2 次提醒降级，延期 3 次必须重拆或暂停。
- 需要读写飞书多维表格时，由 OPC skill 驱动 `lark-base`；不要修改 `lark-base` 的通用规则。

```markdown
- 转介绍：...
- 案例：...

```

## 用户确认边界
必须确认后才可以：
- 追加会议纪要到交付项目文档。
- 更新项目总表。
- 新增今日作战台事项。
- 将会议内容沉淀为客户线索、案例或内容素材。
- 修改项目阶段、风险或状态。

## 与已有 skill 的边界
- `opc-project-rhythm-manager`：负责项目全局节奏；本 skill 负责单次会议到行动项的转化。当输入是会议纪要时，推荐先用本 skill，再二次调用项目节奏 skill。
- `opc-feishu-update-review`：负责每日/每周统一收口；本 skill 生成的拟更新项必须能以结构化清单交由其复核和执行。
- `opc-content-factory-topic-picker` / `opc-content-factory-topic-prioritization`：负责承接会议中出现的客户问题、异议、案例素材和内容选题候选。
- `opc-weekly-review-generator`：负责阶段复盘；本 skill 不写完整复盘。
- `opc-case-asset-archive`：未来负责案例沉淀；本 skill 只识别案例素材机会。

## MVP 验收标准
- 能从会议内容中提取结论、行动项、责任人和时间节点。
- 能判断项目状态、交付物或风险是否变化。
- 能识别是否产生续费、转介绍、案例或内容素材机会。
- 能生成项目总表、今日作战台、交付项目文档的拟更新方案。
- 不新增字段、不擅自写表、不擅自追加文档。

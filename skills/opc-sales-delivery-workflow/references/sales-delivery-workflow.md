# 销售交付链路总览

适用技能：`opc-sales-delivery-workflow`

## 合并后的主链路

```text
线索清空 → 销售对话 → 报价澄清 → 产品推送 → 会议转交付 → 回款/复盘沉淀
```

## 阶段判断

| 阶段 | 关键判断 | 推荐动作 |
|---|---|---|
| 线索进入 | 来源是否可信、需求是否清楚、是否有预算/时限 | 先分类，决定追进/观察/暂停 |
| 初步沟通 | 对方是否有真实痛点和决策权 | 先问问题，不急报价 |
| 需求明确 | 范围、交付物、周期、验收标准是否清楚 | 进入报价澄清 |
| 方案推荐 | 是否匹配现有产品/服务 | 推合适产品，不硬推 |
| 成交后 | 交付动作是否明确，是否影响回款 | 转项目节奏与飞书沉淀 |

## 确认门禁

以下动作必须先给用户拟方案并获得明确确认：

- 正式报价；
- 对外发送客户消息；
- 承诺交付范围、交付周期或价格；
- 修改飞书客户、项目、回款状态；
- 把会议纪要写入正式交付系统。

## 与其他 suite 的关系

- 涉及短期回款、预算、现金缺口：联动 `opc-ceo-rhythm-workflow` 的现金流/机会优先级子流程。
- 已成交、进入交付：联动 `opc-ceo-rhythm-workflow` 的项目节奏子流程。
- 需要写入飞书或沉淀案例：交给 `opc-feishu-update-review`。

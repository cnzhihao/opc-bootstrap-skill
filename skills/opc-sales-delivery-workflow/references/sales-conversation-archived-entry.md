# OPC 销售沟通策略

## 使用场景
当用户面对一个或少量客户，需要判断怎么继续沟通、是否报价、是否暂停、是否发送案例或安排下一步时使用。

典型触发：
- “这个客户下一句怎么回”
- “这个线索要不要继续跟”
- “帮我判断这个客户该不该报价”
- “给我一个推进成交的沟通策略”
- “今天客户跟进怎么排”

## 核心定位
本 skill 是获客技能组的成交推进器，聚焦单个客户或少量客户的沟通策略。它输出可直接执行的下一句话和下一步动作，但不替代收入机会排序、报价范围澄清或线索入池。

P1 阶段只读取复用现有飞书表和文档，不主动新增多维表字段。若用户已确认在 `客户线索` 表增加“线索文档链接/客户线索文档”字段，本 skill 应优先读取对应客户线索文档，承接完整沟通过程。

## 输入信息
优先使用：
- 客户名称或客户线索表记录。
- 最近沟通原文或摘要。
- 客户需求、预算、时间窗口、决策权、信任基础。
- 当前本周主战役。
- 服务产品货架、已有案例、内容材料。

信息不足时，先标记“不足以判断”，并提出需要澄清的问题。不要硬推报价。

## 读取飞书落点
根据需要只读读取：
- `00｜OPC CEO 作战系统 / 客户线索`
- `00｜OPC CEO 作战系统 / 本周主战役`
- `00｜OPC CEO 作战系统 / 今日作战台`
- `00｜OPC CEO 作战系统 / 服务产品货架`
- `05 自媒体内容工厂 / 素材与证据库`，用于寻找可发送的案例、文章或证据
- `02｜周月复盘库`，用于参考历史成交判断和话术资产

## 拟写入飞书落点
所有写入必须先展示拟操作并获得用户确认。

可拟写入：
- 客户线索：更新跟进状态、下一步动作、备注或阶段判断。
- 今日作战台：新增今日或 48 小时客户突破动作。
- 脑内清空池：沉淀新出现的客户需求、产品反馈、内容选题。
- 客户线索文档：补充本次沟通判断、客户异议、报价阻力、下一步动作。引用或更新客户线索文档时，验收顺序必须是：先确认表内链接可打开、文档可读取、内容正确；再检查该链接对应文档是否位于目标客户/线索文件夹。若链接可读但文件夹归位异常，应保留有效链接并提示归位待处理，不重复创建或覆盖文档。
- 内容工厂选题池：将销售过程中的高频问题、典型异议、客户语言、成交/丢单原因桥接给 `opc-content-factory-topic-picker` / `opc-content-factory-topic-prioritization`。

## 工作流程
1. 读取或整理客户上下文。
2. 判断客户阶段：初识、已沟通、需求明确、报价前、待付款、已成交、暂停观察。
3. 识别成交阻力：需求不清、预算不清、时间窗口不明、信任不足、决策权不足、范围不清。
4. 选择推进策略：澄清问题、发案例、约会议、报价、等待、暂停。
5. 生成下一句话，要求可直接复制给客户。
6. 生成下一步动作和截止时间建议。
7. 判断是否需要转交 `opc-quote-scope-clarifier`。
8. 判断本次沟通是否产生可公开表达的内容选题、案例素材或客户异议，并生成内容工厂桥接输入。
9. 生成拟写入飞书方案，并整理成可交给 `opc-feishu-update-review` 的结构化更新清单，等待用户确认。

## 输出格式
默认输出：

```markdown
## 客户阶段判断
- 当前阶段：...
- 判断依据：...

## 当前成交阻力
1. ...

## 建议策略
- 推荐动作：澄清 / 发案例 / 约会议 / 报价 / 暂停
- 判断理由：...

## 可直接发送的话术
> ...

## 下一步动作
| 动作 | 目标 | 建议时间 | 是否写入今日作战台 |
| --- | --- | --- | --- |

## 是否需要转交报价澄清
- 是/否：...
- 原因：...

## 拟写入飞书
- 客户线索：...
- 今日作战台：...
- 脑内清空池：...
- 客户线索文档：...
- 内容工厂：...

## 需要你确认
```

## 用户确认边界
必须确认后才可以：
- 修改客户线索阶段、跟进状态或备注。
- 新增今日作战台动作。
- 将客户标记为暂停/观察。
- 将客户推进到报价、待付款或项目阶段。
- 将客户反馈沉淀为内容选题或产品反馈。

## 与已有 skill 的边界
- `opc-lead-inbox-clearance`：负责线索入池和去重；本 skill 处理已识别客户的沟通策略。
- `opc-income-opportunity-prioritization`：负责多机会排序；本 skill 处理单客户/少量客户的推进策略。
- `opc-quote-scope-clarifier`：负责报价和服务范围；本 skill 判断是否应进入报价澄清。
- `opc-daily-start-workflow`：负责今日整体作战台；本 skill 可提供客户动作输入。
- `opc-feishu-update-review`：负责对拟写入飞书的客户线索、今日作战台、线索文档、内容选题做统一复核与写入。
- `opc-content-factory-topic-picker` / `opc-content-factory-topic-prioritization`：负责承接销售过程中暴露出的选题候选。

## MVP 验收标准
- 能基于客户上下文输出阶段判断。
- 能生成一段可直接发送的话术。
- 能判断应澄清、发案例、约会议、报价或暂停。
- 能生成客户线索和今日作战台的拟更新方案。
- 不新增字段、不擅自写表、不擅自改状态。

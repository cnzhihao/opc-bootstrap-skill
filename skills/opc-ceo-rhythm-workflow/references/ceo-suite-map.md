# CEO 经营节奏 suite 地图

适用主入口：`opc-ceo-rhythm-workflow`

## 主链路

```text
脑内清空 → 事项分流 → 本周主战役 → 现金流生存线 → 机会优先级 → 项目节奏 → 48 小时突破点
```

## 子流程定位

| 子流程 | 原 skill | 当前定位 |
|---|---|---|
| 每日/每周启动 | `opc-daily-start-workflow` | 快捷入口，最终回到 CEO rhythm 主战役判断 |
| 现金流生存线 | `opc-cashflow-survival-check` | 子流程，用于判断最低现金安全线与回款压力 |
| 机会优先级 | `opc-income-opportunity-prioritization` | 子流程，用于判断机会是否进入本周主战役 |
| 项目节奏 | `opc-project-rhythm-manager` | 子流程，用于判断项目推进、暂停、风险和 48 小时动作 |

## 合并后的判断顺序

1. 先清空所有脑内信息，不急着做 todo。
2. 合并同类事项，识别现金流、客户、项目、内容、交付之间的杠杆关系。
3. 只保留本周主战役、必须推进事项、暂停/不做池。
4. 如果现金流紧张，现金流生存线优先级高于普通内容/长期机会。
5. 如果机会很多，按回款确定性、交付成本、战略价值和 48 小时可推进性排序。
6. 最后只选择 1-3 个 48 小时突破点。

## 输出标准

- 本周/今日主战役；
- 必须推进事项；
- 暂停/不做池；
- 现金流或回款风险；
- 48 小时突破点；
- 第一动作。

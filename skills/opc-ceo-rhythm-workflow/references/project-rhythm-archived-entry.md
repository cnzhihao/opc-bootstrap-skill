# OPC 项目节奏管理

## 使用场景
当用户需要判断某个交付项目现在推进到哪里、卡在哪里、今天或 48 小时内应该推进什么时使用。

典型触发：
- “看一下现在项目推进到哪了”
- “这个项目下一步该干什么”
- “帮我梳理交付项目卡点”
- “今天交付侧优先推进什么”
- “这个客户项目有没有风险”

## 核心定位
本 skill 是交付技能组的节奏判断器，负责把项目资料转成可执行的下一步动作。它不是会议纪要工具，也不是周复盘工具。

P1 阶段复用现有 `03｜交付项目` 和 `00｜OPC CEO 作战系统 / 项目总表`，不主动新增多维表字段。若用户输入的是会议纪要、录音转写或客户沟通摘要，应先二次引用 `opc-meeting-summary-to-delivery` 提取会议事实、行动项和交付变化，再由本 skill 判断项目阶段、卡点、风险和下一次关键交付动作。

如需读取或写入飞书多维表格，由本 OPC skill 驱动 `lark-base`；不要修改 `lark-base` 来承载 OPC 业务规则。

## 输入信息
优先使用：
- 项目名称或客户名称。
- 项目总表记录。
- 交付项目文件夹或项目文档。
- 最近会议纪要、交付物、客户反馈。
- 今日作战台与本周主战役。

资料不足时，应列出缺口并提出最小补充问题。

## 读取飞书落点
根据需要只读读取：
- `00｜OPC CEO 作战系统 / 项目总表`
- `00｜OPC CEO 作战系统 / 今日作战台`
- `00｜OPC CEO 作战系统 / 本周主战役`
- `03｜交付项目 / 对应客户或项目文件夹`
- `02｜周月复盘库`，用于参考历史交付经验

## 拟写入飞书落点
所有写入必须先展示拟操作并获得用户确认。

可拟写入：
- 项目总表：阶段、卡点、下一步动作、风险或备注。
- 今日作战台：新增 48 小时内关键交付动作。
- 交付项目文档：追加项目节奏判断或阶段复盘。
- 脑内清空池：沉淀从项目中产生的新经营判断。

## 工作流程
0. 判断输入类型：若是会议纪要、录音转写或客户沟通摘要，先调用 `opc-meeting-summary-to-delivery`，再承接其输出。
1. 明确项目范围和项目资料来源。
2. 读取项目上下文。
3. 判断项目阶段：启动、需求澄清、执行中、待反馈、交付完成、续费判断、暂停、归档。
4. 识别卡点：客户反馈、范围变更、交付物缺口、时间风险、价值感知不足、责任人不清。
5. 判断风险：高、中、低、未知。
6. 生成下一次关键交付动作。
7. 判断是否进入今日作战台。
   - 只有 48 小时内必须推进、且服务当前主战役的项目动作才拟进入今日作战台。
   - 今日作战台每天最多 5 件，超过上限时必须提出合并、降级或暂停建议。
   - 新增项目动作默认状态为「今日待开战」；真正完成用「已完成」；阶段性完成但项目仍继续，用「推进中」。
   - 项目动作延期时，原记录保留原作战日期并设「已延期」，新承接记录使用承接执行日并设「今日待开战」。
   - 承接动作可以改写得更具体，但必须继承或更新「所属主战役」。延期 2 次提醒降级，延期 3 次必须重拆或暂停。
8. 形成拟更新方案，并整理成可交给 `opc-feishu-update-review` 的结构化更新清单，等待确认。

## 输出格式
默认输出：

```markdown
## 项目当前状态一句话

## 阶段判断
- 当前阶段：...
- 判断依据：...

## 当前卡点
1. ...

## 风险判断
- 风险等级：高 / 中 / 低 / 未知
- 原因：...

## 下一次关键交付动作
| 动作 | 目标 | 截止时间 | 是否进入今日作战台 |
| --- | --- | --- | --- |

## 拟写入飞书
- 项目总表：...
- 今日作战台：...
- 交付项目文档：...
- 脑内清空池：...

## 需要你确认
```

## 用户确认边界
必须确认后才可以：
- 修改项目阶段、卡点、风险等级或备注。
- 新增今日作战台交付动作。
- 更新或追加交付项目文档。
- 将项目标记为完成、暂停或归档。
- 将项目产生的信息同步到脑内清空池、内容选题或客户线索。

## 与已有 skill 的边界
- `opc-meeting-summary-to-delivery`：负责先从会议纪要中抽取事实、反馈、行动项和交付物变化；本 skill 负责二次判断项目节奏和风险。当输入是会议纪要时，推荐先用会议转交付 skill，再二次调用本 skill。
- `opc-quote-scope-clarifier`：若卡点来自范围不清，应转回报价/范围澄清。
- `opc-weekly-review-generator`：负责周度复盘；本 skill 处理当下项目推进。
- `opc-daily-start-workflow`：负责今日整体作战台；本 skill 可为其提供交付侧动作。
- `opc-feishu-update-review`：负责统一复核并执行项目总表、今日作战台和项目文档的飞书更新。

## MVP 验收标准
- 能读取或整理一个项目上下文并判断当前阶段。
- 能识别 1-3 个主要卡点。
- 能输出一个 48 小时内可执行的关键交付动作。
- 能生成项目总表、今日作战台、交付文档的拟更新方案。
- 不新增字段、不擅自写表、不擅自归档。

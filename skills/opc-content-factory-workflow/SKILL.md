# OPC Content Factory Workflow

## 目标

将 OPC 的内容生产从零散灵感整理为可交付、可复盘、可迭代的内容流水线，并按「梳理—合并—突破」工作法推进。

## 触发场景

当用户需要：
- 搭建或运行 OPC 内容工厂；
- 批量生产文章、短视频、播客、社群帖、课程内容；
- 从选题到发布制定完整流程；
- 协调多个内容技能或子流程；
- 判断本周/本月内容主战役与 48 小时突破点。

## 输入

可接受以下任意信息：
- 业务目标：增长、转化、品牌、招募、教育、私域运营等；
- 受众画像：人群、痛点、认知阶段、渠道习惯；
- 内容资产：已有文章、选题库、访谈、会议纪要、课程大纲、社群反馈；
- 约束条件：周期、人力、平台、发布频率、禁区、品牌口径；
- 当前问题：缺选题、缺观点、产能低、转化弱、分发乱、复盘缺失。

信息不足时，先用最少问题补齐关键缺口；若用户要求直接开始，则基于显式假设推进。

## 流程

### 1. 梳理：清空信息并转译为事项

- 提取业务目标、核心受众、内容渠道、时间窗口、资源约束。
- 将零散想法转译为可处理事项：选题、素材、观点、内容形态、发布动作、复盘指标。
- 标记不确定项、风险项、依赖项。

### 2. 合并：形成主战役与杠杆动作

- 把事项归并为 1 个本阶段内容主战役。
- 拆出 2-4 个必须推进事项。
- 建立内容链路：选题优先级 → 资料收集 → 观点提炼 → 草稿生成 → 编辑包装 → 分发发布 → 数据复盘。
- 需要判断选题优先级时，调用或转入 `opc-topic-prioritization`。

### 3. 突破：锁定 48 小时行动

- 选择最能撬动结果、阻力最低、反馈最快的突破点。
- 给出 48 小时内的具体交付物、负责人/角色、完成标准与检查点。
- 明确暂停/不做池，避免内容工厂变成长清单。

### 4. 生产编排

按需组织后续子流程：
- 选题筛选与排序；
- 素材收集与洞察提炼；
- 内容大纲与叙事线；
- 初稿、改稿、标题、封面/摘要；
- 多平台改写与发布排期；
- 数据复盘与选题库更新。

## 输出

默认输出结构：

1. **本阶段内容主战役**：一句话说明目标与对象。
2. **已梳理事项**：按选题、素材、生产、分发、复盘归类。
3. **必须推进事项**：2-4 项，说明理由与产出。
4. **暂停/不做池**：当前不值得投入的事项。
5. **48 小时突破点**：明确动作、交付物、完成标准。
6. **下一步生产编排**：建议调用的子技能或下一段流程。

## 边界

- 不替代用户的战略判断；只提供结构化建议与执行编排。
- 不把所有想法都转成待办清单；必须收敛为主战役、杠杆动作和突破点。
- 不编造事实、数据、案例或平台规则；缺少依据时明确标注假设。
- 不输出违法、侵权、虚假营销、隐私泄露或平台违规内容。
- 不直接写入正式技能目录，除非用户明确要求。



## Suite 合并关系

本 skill 是内容工厂 suite 的总入口，负责把选题、访谈、初稿、优化、渠道、视觉、发布、归档和复盘串成一条完整链路。具体子流程由 references 中的 suite 地图分派。

优先合并关系：

- 渠道适配统一由 `opc-content-factory-channel-fit` 承接，旧渠道适配能力已合并进其 references。
- 选题选择统一由 `opc-content-factory-topic-picker` 承接，选题优先级评分能力已合并进其 references。
- 周报和周复盘进入 `opc-content-factory-weekly-report` / 内容工厂周复盘能力，需要飞书归档时交给 `opc-feishu-update-review`。

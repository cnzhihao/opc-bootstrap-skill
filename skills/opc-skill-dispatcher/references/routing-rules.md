# OPC skill 调度路由规则

## 总原则

1. 先判断用户问题属于哪个业务域，而不是直接匹配单个旧 skill 名称。
2. 如果问题跨越多个动作，进入 suite 主入口。
3. 如果问题只涉及保留的专用动作，可进入专用入口。
4. 飞书写入、归档、报价、对外发送、状态迁移等高风险动作必须先确认。

## 主入口优先级

### 1. CEO 经营节奏

优先进入 `opc-ceo-rhythm-workflow`：

- 脑子很乱、事情太多、不知道本周/今天怎么安排；
- 现金流、客户、项目、内容、交付互相冲突；
- 需要每日启动、本周主战役、暂停池、48 小时突破点；
- 需要判断现金流生存线、机会优先级或项目节奏。

### 2. 销售交付

优先进入 `opc-sales-delivery-workflow`：

- 客户线索、销售推进、报价、产品推荐、会议转交付任一出现；
- 用户问“这个客户怎么跟”“怎么报价”“这个需求怎么处理”；
- 销售动作会影响回款、交付范围或项目节奏。

### 3. 内容工厂

优先进入 `opc-content-factory-workflow`：

- 内容想法从选题到发布需要完整推进；
- 同时涉及选题、访谈、初稿、渠道、封面、社交卡片、发布、复盘；
- 用户说初始化/推进/复盘内容工厂。

保留专用入口：

- 只做选题：`opc-content-factory-topic-picker`。
- 只做渠道判断：`opc-content-factory-channel-fit`。
- 只做初稿优化：`opc-content-factory-draft-optimizer`。
- 只做封面/卡片：`opc-content-factory-cover-generator` 或 `opc-content-factory-social-card`。
- 只做 X/Twitter 长文发布：`opc-content-factory-x-article-publisher`。

### 4. 飞书经营系统与周复盘

优先进入 `opc-feishu-update-review`：

- 同步今天工作、更新 OPC 飞书系统、做今天收口；
- 把经营判断、客户线索、内容产物、回款记录沉淀到飞书；
- 需要创建/复制/归档飞书云文档或更新多维表格；
- 需要维护飞书知识库结构。

优先进入 `opc-weekly-review-dispatcher`：

- 做本周复盘、生成周报、串联经营复盘与内容复盘；
- 需要输出周复盘、内容工厂发布稿、长视频口播稿、多平台发布包。

## 确认门禁

以下动作必须先向用户说明拟操作并获得明确确认：

- 实际写入、修改、删除、迁移飞书内容；
- 正式报价或承诺交付范围；
- 对外发送客户消息或发布平台内容；
- 归档、状态迁移、回款记录变更；
- 删除 skill 目录、重命名 skill 包或破坏兼容入口。

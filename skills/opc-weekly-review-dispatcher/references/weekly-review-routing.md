# 周复盘统一调度规则

适用技能：`opc-weekly-review-dispatcher`  
业务域：周复盘

## 作用

本 reference 用于承载 `opc-weekly-review-dispatcher` 中相对稳定、可复用的判断标准、流程清单、模板或验收规则。入口 `SKILL.md` 负责触发条件、执行边界和用户交互；本文件负责沉淀执行时可反复引用的细则。

## 核心使用方式

1. 先根据 `SKILL.md` 判断是否触发该技能。
2. 当任务进入 `周复盘` 相关场景时，读取本 reference 的规则完成判断。
3. 输出时优先保留“结论、依据、下一步动作、需用户确认事项”四类信息。
4. 涉及飞书写入、归档、发布、同步、报价、删除或状态迁移时，先输出拟操作方案并等待用户明确确认。

## 从原技能沉淀出的主要规则索引

- 一、定位
- 二、触发场景
- 三、核心原则
- 1. 先判断，再内容化
- 2. 先经营复盘，再内容复盘
- 3. 最后才生成周报产品
- 4. 写入/发布必须确认
- 四、三段式 Workflow
- 第一段：经营周复盘
- 第二段：内容工厂周复盘
- 第三段：周报内容产品化
- 五、调度判断
- 只做经营复盘时
- 只做内容复盘时
- 只做周报发布稿时
- 做完整周复盘链路时
- 六、标准输出
- 七、与飞书系统的关系

## 交付标准

- 产物要能直接支持用户下一步行动，而不是只复述流程。
- 涉及多个事项时，先合并为主线、主战役或关键路径，再给行动清单。
- 需要外部系统操作时，明确区分“已完成”“拟操作”“待确认”。
- 若输入信息不足，先提出最小必要澄清问题；不因信息不足而编造事实。

当该技能出现稳定的新判断、模板、字段、命名规则或案例时，优先追加到本 reference，而不是继续膨胀 `SKILL.md`。

# Skill: opc-weekly-review-dispatcher

## 一、定位

`opc-weekly-review-dispatcher` 是 OPC 周复盘域的上层路由 Skill。

它负责回答：

> 本周复盘应该按什么顺序跑？哪些结论先沉淀为经营判断，哪些转成内容策略，哪些再变成对外发布稿和长视频口播稿？

它不直接替代具体执行 Skill，而是负责把周复盘任务编排成固定三段式 Workflow：

```text
经营周复盘 → 内容工厂周复盘 → 周报内容产品化
```

对应 Skill：

```text
1. opc-weekly-review-generator
2. opc-content-factory-weekly-review
3. opc-content-factory-weekly-report
```

## 二、触发场景

当用户出现以下任一表达时，优先使用本 Skill：

- “做周复盘”
- “生成本周周报”
- “周末收口”
- “跑完整周复盘链路”
- “本周复盘 + 内容复盘 + 周报”
- “帮我写本周周报/复盘/口播稿”
- “我要经营复盘，也要内容发布稿”
- “需要内容工厂发布稿和长视频口播稿”
- 同时触发 `/opc-weekly-review-generator`、`/opc-content-factory-weekly-review`、`/opc-content-factory-weekly-report` 中两个及以上

## 三、核心原则

### 1. 先判断，再内容化

不得一上来直接写发布稿。必须先形成经营判断：

- 本周主战役是否打赢；
- 现金流和客户线是否推进；
- 哪些成果是真资产，哪些只是噪音；
- 下周唯一主战役是什么；
- 48 小时突破点是什么。

### 2. 先经营复盘，再内容复盘

内容复盘不能脱离经营目标。必须先知道本周经营主战役，才能判断：

- 内容是否服务主战役；
- 哪些内容带来线索/合作/私信/转化；
- 哪些内容只是自嗨或分散注意力；
- 下周内容主战役是否应该跟随经营主战役。

### 3. 最后才生成周报产品

周报发布稿、长视频口播稿、多平台发布包，必须建立在前两段判断之上。

最终发布内容不是流水账，而是围绕一个主线冲突，把一周事件重构成可追更、可传播、可复用的内容产品。

### 4. 写入/发布必须确认

本 Skill 可以生成拟更新方案、周报草稿和发布包；但以下动作必须先获得用户明确确认：

- 写入飞书；
- 修改飞书记录状态；
- 创建/移动/归档飞书文档；
- 发布公众号、X、即刻、小红书等内容；
- 对外发送客户材料。

## 四、三段式 Workflow

### 第一段：经营周复盘

调用：`opc-weekly-review-generator`

目标：形成 CEO 视角的经营判断。

必须产出：

1. **一句话结论**：本周经营状态，不美化；
2. **本周主战役复盘**：原定主战役是什么，是否打赢，为什么；
3. **现金流与客户收入**：收入、回款、线索、客户推进；
4. **内容获客**：本周内容是否带来真实外部信号；
5. **产品/服务/工具资产**：哪些沉淀为资产，哪些只是实验；
6. **系统流程问题**：节奏、飞书、OPC、内容工厂哪里失效；
7. **下周唯一主战役**；
8. **暂停/不做池**；
9. **下周 48 小时突破点**。

输出文件建议：

```text
经营周复盘.md
```

### 第二段：内容工厂周复盘

调用：`opc-content-factory-weekly-review`

目标：判断内容是否服务经营主战役，并确定下周内容策略。

必须产出：

1. **本周内容表现概览**；
2. **有效内容模式**：值得延展的主题、表达、渠道、形式；
3. **失效内容模式**：分散、低效、自嗨、未转化的内容；
4. **内容获客与线索**：私信、合作、客户、转化机会；
5. **可复用内容资产**：工具、Skill、案例、方法论、爆款素材；
6. **未验证假设**；
7. **下周内容主战役**；
8. **48 小时内容突破点**；
9. **内容暂停/不做池**。

输出文件建议：

```text
内容工厂周复盘.md
```

### 第三段：周报内容产品化

调用：`opc-content-factory-weekly-report`

目标：把经营判断和内容判断转译成对外可发布、可视频化、可分发的周报内容产品。

必须产出：

1. **经营周复盘.md**：可同步飞书周/月复盘库；
2. **内容工厂发布稿.md**：面向公众号、知识星球、即刻、X Articles、Substack；
3. **长视频口播稿.md**：面向 B 站、视频号、小红书长视频、播客式口播；
4. **05-发布包/**：长文版、图文版、短版、邮件版等。

## 五、调度判断

### 只做经营复盘时

如果用户只问：

- “本周经营复盘一下”
- “主战役有没有打赢”
- “下周怎么安排”

则只调用：

```text
opc-weekly-review-generator
```

### 只做内容复盘时

如果用户只问：

- “内容这周复盘一下”
- “哪些选题有效”
- “下周内容怎么做”

则只调用：

```text
opc-content-factory-weekly-review
```

### 只做周报发布稿时

如果用户已有明确复盘结论，只要写稿、口播稿或发布包，则调用：

```text
opc-content-factory-weekly-report
```

但如果经营判断缺失，必须先追问或回到第一段。

### 做完整周复盘链路时

如果用户说：

- “做完整周复盘”
- “周复盘 + 内容复盘 + 周报”
- “我要发布稿和长视频口播稿”
- 同时触发三个相关 Skill

则固定执行：

```text
opc-weekly-review-generator
→ opc-content-factory-weekly-review
→ opc-content-factory-weekly-report
```

## 六、标准输出

完整链路结束时，至少输出：

```text
本周一句话结论
本周主战役复盘
本周内容有效/失效模式
下周唯一主战役
下周内容主战役
暂停/不做池
48 小时突破点
周报标题
内容工厂发布稿
长视频口播稿
多平台发布包
飞书拟更新方案（如需要）
```

## 七、与飞书系统的关系

当用户要求“同步飞书”“更新 OPC 系统”“沉淀到周复盘库”时：

1. 先生成拟更新方案；
2. 明确写入哪些表/文档/字段；
3. 等用户确认；
4. 再调用 `opc-feishu-update-review` 或相关飞书工具执行。

不得在未经确认时直接写入、改状态、归档或发布。

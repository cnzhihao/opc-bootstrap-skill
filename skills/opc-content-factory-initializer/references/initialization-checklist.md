# OPC 内容工厂初始化技能

## 一、技能定位

本技能负责在飞书云盘中初始化「05 自媒体内容工厂」工作空间。

它是 OPC 自媒体技能组的基础设施技能，不负责写文章、发内容或做选题判断，而是为后续内容生产链路提供稳定目录和模板落点。

一句话定位：

> 在飞书云盘里搭好 OPC 自媒体内容工厂的地基，让选题、素材、稿件、发布、复盘和资产沉淀都有固定位置。

## 二、触发场景

当用户提出以下需求时，使用本技能：

- 初始化内容工厂；
- 新建 05 自媒体内容工厂；
- 在飞书云盘创建内容工厂目录；
- 搭建自媒体技能组文件夹；
- 补齐内容工厂目录结构；
- 重建/检查内容工厂初始化状态；
- 为 `opc-content-factory-*` 技能创建工作空间。

## 三、核心原则

1. **只新增和补齐，不删除已有内容。**
2. **不覆盖已有文档。**如目标文件已存在，应先提示用户，或创建缺失项。
3. **按内容生命周期建目录，不按 skill 名称建目录。**目录应该服务长期使用，而不是服务某个工具。
4. **内容工厂属于自媒体技能组，不单独作为一级业务技能组。**
5. **初始化只是搭地基，不直接生成文章、不发布内容、不移动旧资料。**

## 七、执行流程

### 1. 确认目标位置

先确认飞书云盘父目录。如果用户明确给出目录，则使用用户指定目录。

### 2. 检查是否已有「05 自媒体内容工厂」

- 如果不存在：创建完整目录结构。
- 如果存在：只补齐缺失目录和缺失文档。
- 如果发现重名多个目录：停下并让用户确认使用哪个。

### 3. 创建顶层目录和子目录

先创建顶层目录，再创建一级和二级目录。

### 4. 创建核心文档

创建 README、看板、排产表、暂停池和 3 个模板。

### 5. 返回初始化结果

最终反馈应包含：

- 顶层目录名称和飞书链接；
- 已创建的一级目录；
- 已创建的核心文档；
- 是否有跳过项；
- 下一步建议：启动 `opc-content-factory-workflow` 做第一次内容主战役梳理。

## 八、禁止行为

本技能禁止：

- 删除已有飞书文件或文件夹；
- 覆盖已有文档；
- 移动用户已有资料；
- 把内容工厂作为独立于自媒体技能组的一级业务技能组；
- 初始化时直接生成文章；
- 初始化时直接发布内容；
- 在没有用户确认的情况下清空任何选题库、素材库或稿件库。

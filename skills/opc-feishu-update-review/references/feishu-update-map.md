# OPC 多维表格更新映射

本文件是 `opc-feishu-update-review` 的执行参考，用于判断一条经营信息应该如何从「拟进入脑内清空池」流转到各业务表。

## 最高规则：确认前不得写表

任何用户新增信息，在用户明确确认前，只能形成「拟更新方案」，不得实际新增、修改、删除、归档或迁移多维表格记录。

确认前允许：读取表结构、读取相关记录、整理原话、做经营判断、输出拟更新方案。

确认后才允许：按确认范围写入脑内清空池、主战役、客户线索、内容选题池、服务产品货架、项目总表、月度现金流、回款记录、周/月复盘素材。

删除、归档、标记完成、标记废弃、退出脑内清空池、关闭项目、取消客户线索、把现金流改为已到账，都必须单独确认。

## 总入口：拟进入脑内清空池

所有新增经营信息默认先拟进入「脑内清空池」。

拟记录字段建议：

- 原始信息：保留用户原话。
- 转译事项：转成可判断事项。
- 信息类型：客户 / 项目 / 内容 / 产品 / 现金流 / 主战役 / 风险 / 复盘 / 暂停。
- 当前判断：它可能意味着什么。
- 是否命中本周主战役：是 / 否 / 待确认。
- 可能去向：主战役 / 客户线索 / 内容选题 / 服务产品 / 项目总表 / 回款记录 / 暂停 / 归档 / 废弃。
- 下一步动作：需要谁做什么。
- 状态建议：新进入 / 待判断 / 已分流 / 暂停 / 已归档 / 废弃。

## 主战役判断

命中主战役的信号：

- 直接推进本周最重要经营目标。
- 影响客户成交、现金流、交付、内容获客或产品验证。
- 是本周必须突破的关键阻塞。
- 与当前 48 小时突破点有关。

拟更新主战役时说明：

- 关联哪一个主战役。
- 是进展、风险、阻塞、下一步还是复盘素材。
- 对本周结果的影响。
- 下一步动作。

不命中主战役时说明：

- 为什么不命中。
- 是否进入暂停池、观察池、内容储备、客户储备或归档。

## 客户线索判断

形成客户线索的信号：

- 出现明确外部对象：客户、机构、街道、园区、企业、个人。
- 出现合作动作：介绍信、拜访、沟通、需求、报价、方案、合同、付款。
- 出现预算、决策人、使用场景或成交可能。

拟更新客户线索时至少说明：

- 客户名称。
- 当前阶段。
- 需求/机会描述。
- 下一步动作。
- 关联主战役。
- 是否已有相关记录，拟新增还是拟修改。

## 内容选题判断

形成内容选题的信号：

- 可公开表达的经营判断。
- 客户问题可转化为内容。
- 创始人真实经历有信任建设价值。
- 可沉淀方法论、案例、复盘、观点。
- 有获客、转化、品牌或资产价值。

拟更新内容选题时至少说明：

- 选题标题。
- 核心观点。
- 服务哪个经营目标。
- 适合渠道。
- 优先级：P0 / P1 / P2 / P3。
- 下一步内容动作。

## 服务产品判断

形成服务产品的信号：

- 同类客户反复提出相似需求。
- 可被标准化、打包、报价、交付。
- 具备明确服务对象、交付物、周期和价值。
- 是从项目经验中抽象出的可复用能力。

拟更新服务产品时至少说明：

- 产品/服务名称。
- 目标客户。
- 痛点。
- 核心交付物。
- 当前阶段：想法 / 验证 / 可售 / 交付中 / 暂停。
- 是否应该现在推。

## 项目总表判断

需要更新项目总表的信号：

- 线索进入正式项目。
- 项目阶段变化：待报价、已报价、已成交、交付中、验收、暂停、关闭。
- 范围、周期、负责人、报价、交付物、风险发生变化。
- 客户线索已经变成可管理项目。

拟更新项目总表时至少说明：

- 项目名称。
- 客户/关联对象。
- 当前阶段。
- 项目范围。
- 下一步动作。
- 风险或阻塞。
- 是否影响现金流。

## 月度现金流 / 回款记录判断

需要更新现金流或回款记录的信号：

- 已报价。
- 已确认付款意向。
- 已签约。
- 已到账。
- 部分到账。
- 待回款。
- 回款延迟。
- 退款、坏账或成本变化。
- 月度现金缺口发生变化。

拟更新时至少说明：

- 现金流类型：报价 / 待回款 / 已到账 / 成本 / 风险。
- 客户或项目。
- 金额：如未知则标记待确认，不能编造。
- 预计时间。
- 当前状态。
- 是否影响月度现金流判断。

## 脑内清空池退出条件

清空池不能只进不出。

可退出条件：

- 已进入对应业务表，并有明确下一步动作。
- 已确认不服务当前主战役，进入暂停/不做池。
- 已沉淀为周/月复盘或方法文档。
- 已确认重复、过期、无行动价值，进入废弃。
- 已完成且无需继续跟踪。

退出时必须说明：

- 退出状态。
- 退出去向。
- 判断依据。
- 关联记录。
- 是否需要继续复盘。

退出、归档、废弃、删除都必须单独获得用户确认。

## 自检清单

提交拟更新方案前，检查：

1. 是否先拟进入脑内清空池？
2. 是否明确没有在确认前写表？
3. 是否判断了本周主战役关系？
4. 是否判断了客户线索？
5. 是否判断了内容选题？
6. 是否判断了服务产品？
7. 是否判断了项目总表？
8. 是否判断了现金流和回款记录？
9. 是否给出清空池退出条件？
10. 是否列出需要用户确认的问题？
11. 是否把删除、归档、退出、状态迁移单独列出？

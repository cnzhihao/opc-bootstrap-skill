# 脑内清空池与分流规则

适用技能：`opc-feishu-update-review`  
业务域：飞书经营系统

> 本文件由原 `SKILL.md` 中稳定、可复用的规则章节迁移而来。入口 `SKILL.md` 仅保留触发、流程骨架和引用关系。

## 脑内清空池规则

### 进入条件

用户给出的信息，无论是确定事实、模糊想法、机会、焦虑、客户进展、现金流变化、项目变化、内容灵感，都应先拟进入「脑内清空池」。

确认前只是拟进入，不代表已经写表。

拟写入时至少保留：

- 原始信息：尽量保留用户原话。
- 转译事项：把原话转成可执行/可判断事项。
- 信息类型：客户 / 项目 / 内容 / 产品 / 现金流 / 主战役 / 风险 / 复盘 / 暂停。
- 当前判断：初步判断它可能意味着什么。
- 下一步动作：如果还不能分流，下一步需要确认什么。
- 进入时间：当天日期。
- 状态：新进入 / 待判断 / 已分流 / 已归档 / 暂停 / 废弃。

### 状态机

推荐状态：

1. 新进入：刚从聊天中捕获，尚未判断。
2. 待判断：需要补充信息才能分流。
3. 已分流：已经进入客户线索、内容选题、服务产品、项目总表、回款记录或主战役。
4. 暂停：当前不推进，但未来可能有价值。
5. 已归档：完成复盘沉淀，短期不再追踪。
6. 废弃：判断无价值或不符合当前战略。

### 退出去向

清空池事项只能退出到以下去向之一或多个：

- 本周主战役：直接服务本周最重要目标。
- 客户线索：存在明确外部对象和商业机会。
- 内容选题池：具备表达、获客、信任建设或方法沉淀价值。
- 服务产品货架：可包装为可重复售卖服务。
- 项目总表：进入报价、成交、交付、验收或项目管理。
- 回款记录/月度现金流：产生到账、待回款、报价、成本或缺口变化。
- 暂停/不做池：有诱惑但不服务当前主战役。
- 周复盘/月复盘素材：形成关键经营判断，但不需要持续跟进。
- 废弃：无行动价值、重复、过期或噪音。

### 退出判定

一条清空池事项可以退出，当且仅当满足以下至少一项：

- 已经形成明确下一步动作，并进入对应业务表。
- 已被判断为不服务当前主战役，并进入暂停/不做池。
- 已经沉淀为复盘判断或方法文档。
- 已经确认无价值、重复或过期，可以废弃。
- 已经完成，不需要再跟踪。

退出时必须记录：

- 退出状态。
- 退出去向。
- 判断依据。
- 关联记录，如客户线索、项目、选题、产品、回款。
- 是否需要复盘。

注意：退出、归档、废弃必须在拟更新方案中单独列出，并获得用户确认。

## 本周主战役判断

读取或参考「本周主战役」表，判断清空池事项是否命中本周主战役。

判断标准：

- 是否直接服务本周现金流突破。
- 是否推进本周最重要客户、渠道或交付。
- 是否服务本周 48 小时突破点。
- 是否影响本周主战役完成标准。
- 是否暴露本周主战役的新风险或新机会。

如果命中，拟更新方案中应说明：

- 清空池中拟标注：命中本周主战役 = 是。
- 拟关联的主战役名称。
- 拟同步到主战役的进展、风险、下一步或复盘素材。

如果不命中，拟更新方案中应说明：

- 清空池中拟标注：命中本周主战役 = 否。
- 不命中原因。
- 是否进入暂停/不做池，或作为资产储备。

不要因为信息存在就自动提升为主战役。

## 跨表派生判断

### 客户线索

满足任一条件，可建议创建或更新「客户线索」：

- 出现明确客户、街道、园区、机构、企业、渠道、合作方或联系人。
- 出现介绍信、拜访、报价、方案、需求沟通、转介绍、合作意向。
- 出现可以带来收入或合作的外部对象。
- 出现需要持续跟进的潜在成交机会。

拟更新客户线索时至少判断：

- 客户/机构名称。
- 当前阶段：初识 / 已接触 / 介绍信已提交 / 需求沟通 / 方案中 / 报价中 / 待成交 / 已成交 / 暂停。
- 成交概率或优先级。
- 下一步动作。
- 是否关联本周主战役。
- 是否关联项目总表。
- 来源清空池事项。

### 内容选题池

满足任一条件，可建议创建或更新「内容选题池」：

- 用户表达了观点、判断、案例、复盘、方法论、金句。
- 某个客户、项目或战役过程具有可公开表达价值。
- 能帮助获客、解释服务、建立信任或沉淀 OPC 方法。
- 可以转化为文章、小报、蓝皮书、招商页、短视频脚本或朋友圈素材。

拟更新内容选题时至少判断：

- 选题标题或方向。
- 内容来源：客户推进 / 项目复盘 / 主战役 / 个人判断 / 方法沉淀。
- 经营相关性：是否服务现金流、客户转化、产品解释。
- 优先级：P0 48 小时突破 / P1 本周主战役 / P2 资产储备 / P3 暂停。
- 下一步动作：访谈 / 起草 / 发布 / 暂存。
- 是否关联本周主战役。

### 服务产品货架

满足任一条件，可建议创建或更新「服务产品货架」：

- 出现可重复交付的服务形态。
- 某个客户需求可以包装成标准服务。
- 项目交付中沉淀出模板、流程、工具、训练营、咨询包、陪跑包。
- 市场反复出现同类需求。
- 内容资产可以承接为可售产品。

拟更新服务产品时至少判断：

- 产品/服务名称。
- 对应客户痛点。
- 当前状态：想法 / 内部验证 / 可试卖 / 正在卖 / 暂停。
- 是否应该现在推：立刻售卖 / 渠道测试 / 内容资产 / 内部使用 / 暂停。
- 价格或报价假设。
- 下一步动作。
- 来源清空池事项。

### 月度现金流与回款记录

战役推进或打完后，必须判断是否产生现金流变化。

现金流变化包括：

- 已到账：真实收到的钱。
- 待回款：已确认或大概率收款但尚未到账。
- 报价中：已给出价格但尚未确认。
- 成本变化：新增支出、固定成本、社保、公积金、工资、软件、外包。
- 缺口变化：本月现金流缺口扩大或缩小。

如果出现真实新收入或回款，可建议更新「回款记录」。

回款记录必须区分：

- 已到账：实际收到的钱，必须有金额、日期、付款方或来源。
- 待回款：不能写成已到账，必须标注预计金额、预计时间、触发条件。
- 报价/意向：不能写成待回款，除非客户已经明确确认。

同时根据需要建议更新「月度现金流」：

- 当月已到账收入。
- 当月待回款。
- 当月硬现金支出。
- 当月缺口。
- 对收入目标、社保、公积金、固定成本覆盖情况的影响。

### 项目总表

满足任一条件，可建议更新「项目总表」：

- 客户线索进入报价、成交、交付或复购。
- 已有项目的范围、时间、交付物、负责人、报价、状态发生变化。
- 战役推进导致项目优先级变化。
- 项目暂停、延期、取消、完成。
- 出现新的交付风险或验收节点。

拟更新项目总表时至少判断：

- 项目名称。
- 关联客户。
- 当前状态：机会 / 报价 / 已成交 / 进行中 / 待验收 / 已完成 / 暂停 / 取消。
- 交付范围。
- 金额或报价。
- 下一步动作。
- 风险。
- 是否关联本周主战役。
- 来源清空池事项。

## 字段处理原则

更新时不要追求一次性字段完美，优先保证：

1. 事实被记录。
2. 判断被记录。
3. 下一步动作明确。
4. 状态可追踪。
5. 金额、时间、客户、项目尽量量化。
6. 关联关系可追溯。

如果字段不存在，不要臆造已经写入；应使用现有字段近似记录，或提示需要补字段。

如果字段含义不确定，必须先问用户或在拟更新方案中列为待确认。

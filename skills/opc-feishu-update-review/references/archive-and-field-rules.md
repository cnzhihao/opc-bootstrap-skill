# 归档命名与字段处理规则

适用技能：`opc-feishu-update-review`  
业务域：飞书经营系统

> 本文件由原 `SKILL.md` 中稳定、可复用的规则章节迁移而来。入口 `SKILL.md` 仅保留触发、流程骨架和引用关系。

## 固定飞书结构

默认 OPC 系统结构为：

```text
OPC秘书
├── 00｜OPC CEO 作战系统（多维表格）
├── 01｜秘书工作方法
├── 02｜周月复盘库
│   └── YYYY-MM
│       ├── YYYY-MM-W1｜周复盘｜完整模板
│       ├── YYYY-MM-W2｜周复盘｜完整模板
│       ├── YYYY-MM-W3｜周复盘｜完整模板
│       ├── YYYY-MM-W4｜周复盘｜完整模板
│       └── YYYY-MM｜月度完整复盘｜完整模板
└── 99｜归档与暂停资料
```

OPC CEO 作战系统应包含以下表：

- 脑内清空池
- 本周主战役
- 月度现金流
- 客户线索
- 服务产品货架
- 内容选题池
- 项目总表
- 回款记录

## 拟更新方案格式

在未获得确认前，输出必须使用「拟更新方案」而不是「已更新」。

```text
我建议这样更新，请确认：

一、拟进入脑内清空池
- 原始信息：……
- 转译事项：……
- 信息类型：客户 / 项目 / 内容 / 产品 / 现金流 / 主战役 / 风险 / 复盘 / 暂停
- 当前判断：……
- 下一步动作：……
- 状态建议：新进入 / 待判断

二、是否命中本周主战役
- 判断：是 / 否 / 待确认
- 关联主战役：……
- 判断依据：……
- 拟更新动作：……

三、拟派生更新
1. 客户线索：写 / 不写 / 待确认
   - 拟字段：……
   - 判断依据：……
2. 内容选题池：写 / 不写 / 待确认
   - 拟字段：……
   - 判断依据：……
3. 服务产品货架：写 / 不写 / 待确认
   - 拟字段：……
   - 判断依据：……
4. 项目总表：写 / 不写 / 待确认
   - 拟字段：……
   - 判断依据：……
5. 月度现金流/回款记录：写 / 不写 / 待确认
   - 拟字段：……
   - 判断依据：……

四、拟清空池退出/归档/删除
- 是否涉及退出：是 / 否 / 待确认
- 退出去向：……
- 是否涉及归档：……

五、需要你确认的问题
1. ……
2. ……

请确认是否按以上方案写入。未确认前我不会操作多维表格。
```

## 周复盘生成

当用户说“做本周复盘”时，按 `references/weekly-review-template.md` 生成复盘。

周复盘不是流水账，必须包含：

1. 本周一句话结论。
2. 本周主战役及是否切换。
3. 清空池中哪些事项被正确分流，哪些仍然堆积。
4. 现金流进展。
5. 客户与收入机会。
6. 内容与获客。
7. 产品与服务判断。
8. 项目交付和风险。
9. 系统与流程复盘。
10. 下周 48 小时突破点。

## 月复盘生成

当用户说“做月度复盘”或到月底时，按 `references/monthly-review-template.md` 生成复盘。

月复盘必须回答：

- 是否覆盖社保、公积金、固定成本。
- 是否达成月度收入目标。
- 收入来自哪里。
- 哪些动作 ROI 最高。
- 哪些服务值得继续卖。
- 哪些机会应该暂停。
- 哪些清空池事项反复堆积，说明系统哪里有问题。
- 下月主战役是什么。

# 周报生成全流程（主线冲突式 + 追更机制）

把一周碎片化的事件，重构成有故事感、可追更的周报内容产品。核心逻辑与内容工厂完全一致——访谈 → 主线提炼 → 5 轮优化 → 周复盘/发布稿/口播稿 → 发布包，只是输入从"选题"换成"本周事件"，输出从"干货文"换成"连载式周报"。

## 7 步工作流

### 步骤 1：Vault 自动扫描 + 创建工作空间

**1.1 扫描本周客观成果**（让 AI 帮你回忆，减轻访谈负担）：

- 扫描 `02-稿件库/01-创作中稿件/` 找出本周创建/修改的稿件文件夹（按文件 mtime 过滤最近 7 天）
- 扫描 `05-数据复盘库/` 找出本周发布文章的数据复盘
- 扫描 `03-素材库/` 统计本周新增素材数量
- 把扫描结果汇总成「本周客观成果清单」，告诉用户

**1.1b 补齐周复盘 / 发布稿 / 长视频所需经营素材**：

除内容成果外，还必须补齐以下信息；信息缺失时用访谈追问补齐：

- 本周原定主战役、是否打赢、失败/成功原因；
- 本周现金流、回款、客户线索、重要商务推进；
- 本周内容获客信号：播放、阅读、评论、私信、转发、线索；
- 本周沉淀的产品、服务、工具、Skill、方法论资产；
- 本周系统/流程失效点：为什么发散、哪里被打断、哪个机制没执行；
- 下周唯一主战役、暂停池、48 小时突破点；
- 长视频素材：最有冲突感的一句话、能讲出来的具体场景、可展示的画面/截图/屏幕录制、适合切片的金句。

**1.2 计算本期编号和文件夹名**：

```bash
# 自动计算下一期编号
WEEKLY_DIR="weekly-reports"
YEAR=$(date +%Y)
WEEK_NUM=$(date +%V)
# 扫描已有周报数量 +1 作为本期编号（#001 起步，与 ISO 周数不同）
EXISTING_COUNT=$(ls -d "${WEEKLY_DIR}/${YEAR}/"W*-*/ 2>/dev/null | wc -l | tr -d ' ')
ISSUE_NUM=$(printf "%03d" $((EXISTING_COUNT + 1)))

# 等用户确认主线标题后再创建文件夹，先准备好编号
echo "本期编号：#${ISSUE_NUM}（${YEAR}-W${WEEK_NUM}）"
```

**1.3 启动逐题动态访谈**：

读取 `references/interview-flow.md` 执行访谈。**这是本 Skill 最关键的环节**，按以下动作标准推进：

- **单轮单问**：每次提出 1 个开放问题，收到用户回答后再进入下一问。
- **开放问答**：用自然对话承接用户回答，围绕真实经历、关键判断和具体画面展开。
- **动态追问**：根据用户回答决定下一个问题，让访谈沿着最有张力的线索推进。
- **追问触发条件**：回答过短（< 2 句）、有趣瞬间尚未展开、回答偏抽象缺画面、关键信息一笔带过——出现这些情况立即追问 1 次。
- **访谈节奏**：5-8 题，全程 10-15 分钟，聚焦周报所需的主线、事件、反思和下周悬念。
- **5 阶段推进路径**：一周倾倒 → 主线挖掘 → 画面补全 → 反思钩子 → 下周悬念。
- **结束判断**：同时满足主线已挖、事件有画面、反思有钩子、下周有悬念 4 个条件后主动结束。

访谈阶段先累积问答原文和关键判断。访谈结束后，再创建工作空间并一次性写入 `01-访谈记录.md`。

### 步骤 2：主线冲突提炼（关键步骤，决定整篇周报的故事性）

读取 `references/storyline-conflict.md` 中的提炼逻辑：

1. 访谈结束后，AI **只抛 1 个最强主线候选**，用 50 字内复述给用户：

```
听完你这周的描述，我想把这期的主线定为：

「[主线冲突复述，50 字内]」

理由：[一句话说明为什么这个主线最有张力]

你看这个方向对吗？或者你想改成什么？
```

2. **等用户回复**：用户确认 / 改写 / 重新挖
3. 用户确认后，把主线标题作为文件夹名的一部分：

```bash
# 主线标题简化（去空格、去特殊字符，保留中文）
SLUG="删了3篇10w+的稿子"  # 用户确认后的主线短句
FOLDER_NAME="W${WEEK_NUM}-${SLUG}"
FOLDER_PATH="weekly-reports/${YEAR}/${FOLDER_NAME}"

mkdir -p "${FOLDER_PATH}/05-发布包"
# 创建工作文件（访谈记录已经在内存中，直接写入）
```

5. 把访谈记录、主线冲突、本期编号一起写入 `01-访谈记录.md`

### 步骤 3：初稿生成 + 5 轮迭代优化

**3.0 生成初稿**：

读取 `references/prompt-draft-weekly.md` 中的初稿生成 Prompt：
- 严格按 `references/template-weekly-structure.md` 的结构生成
- 所有事件、数字、人物、结果都能对应到访谈记录或扫描结果；缺失信息用 `[待补充]` 标记并回问
- 风格 100% 贴合主风格报告
- 主线冲突必须出现在标题 + 开篇 100 字内
- 写入 `02-初稿.md`

**3.1 - 3.5 五轮优化（魔改 draft-optimizer）**：

读取 `references/prompt-rounds-1-5-weekly.md`，按以下顺序执行：

| 轮次 | 优化维度 | 周报特有关切点 |
|------|---------|--------------|
| 第 1 轮 | 主线贯穿度 | 每个段落都呼应主线冲突，保留服务主线的内容 |
| 第 2 轮 | 故事张力 | 开场 100 字是否抓人？转折是否有画面感？事件是否具体？ |
| 第 3 轮 | 语言风格 | 贴合主风格报告；周报允许更口语、更"不确定"的表达 |
| 第 4 轮 | 追更感注入 | 数据仪表盘清晰、下周预告留悬念、本周一问能引互动 |
| 第 5 轮 | 定型 + 清 AI 味 | 改写二元对立模板、工整三段排比等 AI 痕迹，形成自然表达 |

每轮覆盖 `02-初稿.md`，追加 `03-改稿记录.md`，第 4 轮起写入 `04-定稿.md`。

### 步骤 4：AI 味扫描（推荐）

调用 `/dbs-ai-check` 对 `04-定稿.md` 做扫描，仅扫描不修改。如有严重 AI 味特征，回到第 3 轮修复。

### 步骤 5：注入追更元素（周报独有）

读取 `references/recurring-elements.md`，在 `04-定稿.md` 末尾或对应位置注入：

1. **标题加编号**：`智昊周报 #012｜这周我删了3篇10万+的稿子`
2. **本周一图**：留 `![[占位]]` 让用户后续补图（或调用 `/cover-generator`）
3. **数据仪表盘**（自动填充可计算字段）：
   - 💰 收入/变现：[访谈第 3 题回答]
   - 📚 本周学到的一句话：[访谈第 6 题"意外"中提取]
   - 📊 内容产出：完成 X 篇 / 发布 X 篇（来自 Step 1 扫描结果）
4. **下周预告**：从访谈「下周悬念」阶段问答改写为悬念句，控制在 30 字内
5. **本周一问**：基于本周主线生成 1 个开放性问题，引导读者评论

### 步骤 6：生成周复盘 + 内容工厂发布稿 + 长视频口播稿 + 多平台发布包

本步骤是周报的固定交付结构。除非用户明确只要其中一种，否则必须同时产出以下 4 类文件。

#### 6.1 经营周复盘：`经营周复盘.md`

用途：同步到 OPC CEO 作战系统 / 飞书周月复盘库。它定位为经营判断文档，服务主战役复盘、资源取舍和下周突破。

必须包含：

1. **一句话结论**：本周经营判断，表达真实状态和核心判断；
2. **本周主战役复盘**：原定主战役是什么，是否打赢，失败/成功原因；
3. **现金流与客户收入**：本周收入、回款、线索、客户推进；暂无进展时写明“无新增/未推进”；
4. **内容获客与外部信号**：本周发布、爆款、私信、线索、值得延展的选题；
5. **产品/服务/工具资产**：本周沉淀了哪些可复用资产，哪些只是实验；
6. **系统流程问题**：OPC 系统、内容工厂、个人节奏哪里失效；
7. **下周主战役**：只能有一个主战役，说明为什么；
8. **暂停池**：明确下周收敛边界，让主战役获得足够注意力；
9. **下周 48 小时突破点**：一个能在 48 小时内推进的最小突破。

#### 6.2 内容工厂发布稿：`内容工厂发布稿.md`

用途：公众号、知识星球、即刻、X Articles、Substack 等长文渠道。它要从经营复盘转译成读者愿意看的**长文章/专栏周报**，以完整阅读体验承载本周主线、关键事件和创始人判断。

必须采用长文章形态：

1. **标题**：统一使用 `智昊的一人公司周报 NN｜[本期主线冲突短句]`，保留主线冲突和系列编号；
2. **导语 150-250 字**：用完整段落抛出本周最大冲突、读者收益和继续阅读理由；
3. **4-6 个二级标题分节**：参考 W3《内容工厂发布稿》的形态，每节围绕一个判断展开；
4. **每节 2-4 个自然段**：段落包含论述、背景、事件、判断和转折，形成长文章阅读节奏；
5. **本周发生了什么**：选择 3-5 个关键事件，围绕主线重排；
6. **真正的问题是什么**：从事件上升到创始人/内容生产/AI 工作流的普遍问题；
7. **本周有效资产**：把成果讲成可复用资产，说明后续如何继续发挥价值；
8. **本周失败判断**：承认真实失败，并提炼可复用洞察；
9. **下周主战役**：给读者一个可追更的悬念；
10. **本周一问**：一个开放问题，引导评论/私信。

长文章验收标准：

- 全文由主线冲突贯穿，并通过二级标题组织阅读节奏；
- 正文采用长文章/公众号主稿结构，包含完整自然段、论述层次和创始人判断；
- 关键事件按主线权重安排篇幅，重点事件展开背景、动作、结果和反思；
- 至少有一段能被单独截图传播的长文判断；
- 结尾留下“下周我会继续追这条线”的追更感；
- 生成后自检：二级标题完整、段落有展开、读起来像专栏文章，即可交付为 `内容工厂发布稿.md`。

#### 6.3 长视频口播稿：`长视频口播稿.md`

用途：B 站、视频号、小红书长视频、播客式口播。目标时长通常 5-12 分钟。

必须包含：

1. **视频标题候选**：3 个，分别偏真实复盘/传播钩子/方法论；
2. **开场 15 秒 Hook**：一句话说出冲突，例如“我这周做了很多事，但我认为这周在工作上是失败的”；
3. **口播正文**：按自然说话方式写，使用适合镜头前表达的短句、停顿和转折；
4. **段落节奏标记**：每 60-90 秒一个小节，标注 `[转场]`、`[停顿]`、`[强调]`；
5. **画面建议**：每个小节给 1-2 个可配画面/屏幕录制/截图建议；
6. **金句切片**：3-5 句可单独剪成短视频的表达；
7. **结尾 CTA**：引导评论、私信、关注下一期或领取相关资料。

长视频口播稿推荐结构：

```md
# 长视频口播稿｜[标题]

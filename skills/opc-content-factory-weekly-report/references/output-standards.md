## 核心使用方式

1. 先根据 `SKILL.md` 判断是否触发该技能。
2. 当任务进入 `内容工厂周报` 相关场景时，读取本 reference 的规则完成判断。
3. 输出时优先保留“结论、依据、下一步动作、需用户确认事项”四类信息。
4. 涉及飞书写入、归档、发布、同步、报价、删除或状态迁移时，先输出拟操作方案并等待用户明确确认。

## 交付标准

- 产物要能直接支持用户下一步行动，而不是只复述流程。
- 涉及多个事项时，先合并为主线、主战役或关键路径，再给行动清单。
- 需要外部系统操作时，明确区分“已完成”“拟操作”“待确认”。
- 若输入信息不足，先提出最小必要澄清问题；不因信息不足而编造事实。

# 周报生成全流程（主线冲突式 + 追更机制）

把一周碎片化的事件，重构成有故事感、可追更的周报内容产品。核心逻辑与内容工厂完全一致——访谈 → 主线提炼 → 5 轮优化 → 发布包，只是输入从"选题"换成"本周事件"，输出从"干货文"换成"连载式周报"。

## 前置条件

- 已有写作风格报告：`00-模板库/作者个人写作风格分析报告.md`
- 已有自媒体账号基础信息：`00-模板库/自媒体账号基础信息.md`
- 周报目录 `weekly-reports/`（不存在时本 Skill 会自动创建）

## 7 步工作流

### 步骤 1：Vault 自动扫描 + 创建工作空间

**1.1 扫描本周客观成果**（让 AI 帮你回忆，减轻访谈负担）：

- 扫描 `02-稿件库/01-创作中稿件/` 找出本周创建/修改的稿件文件夹（按文件 mtime 过滤最近 7 天）
- 扫描 `05-数据复盘库/` 找出本周发布文章的数据复盘
- 扫描 `03-素材库/` 统计本周新增素材数量
- 把扫描结果汇总成「本周客观成果清单」，告诉用户

**1.2 计算本期编号和文件夹名**：

```bash
# 自动计算下一期编号
WEEKLY_DIR="weekly-reports"
YEAR=$(date +%Y)
WEEK_NUM=$(date +%V)
# 扫描已有周报数量 +1 作为本期编号（#001 起步，与 ISO 周数不同）
EXISTING_COUNT=$(ls -d "${WEEKLY_DIR}/${YEAR}/"W*-*/ 2>/dev/null | wc -l | tr -d ' ')
ISSUE_NUM=$(printf "%03d" $((EXISTING_COUNT + 1)))

# 等用户确认主线标题后再创建文件夹，先准备好编号
echo "本期编号：#${ISSUE_NUM}（${YEAR}-W${WEEK_NUM}）"
```

**1.3 启动逐题动态访谈**：

读取 `references/interview-flow.md` 执行访谈。**这是本 Skill 最关键、最不可妥协的环节**，必须严格遵守以下规则：

- **每次只问 1 个问题**，等用户回答后再问下一个
- **绝对禁止一次性抛出多题清单**（这是把访谈做成"清单填空"的根本错误）
- **绝对禁止使用 AskUserQuestion 之类的多选菜单**替代开放式问答
- **动态追问**：根据用户回答决定下一个问题，不按预设列表机械提问
- **追问触发条件**：回答过短（< 2 句）、有趣瞬间未展开、回答偏抽象缺画面、关键信息一笔带过——出现这些情况立即追问 1 次
- **访谈节奏**：5-8 题，全程 10-15 分钟，比 /content-interview 轻（周报不需要"行业认知升华"层）
- **5 阶段推进路径**：一周倾倒 → 主线挖掘 → 画面补全 → 反思钩子 → 下周悬念
- **结束判断**：4 个条件都满足（主线已挖、事件有画面、反思有钩子、下周有悬念）才主动结束

访谈进行中不要急着创建文件夹或写文件，把问答原文先在内存里累积。访谈结束后再一次性写入 `01-访谈记录.md`。

### 步骤 2：主线冲突提炼（关键步骤，决定整篇周报的故事性）

读取 `references/storyline-conflict.md` 中的提炼逻辑：

1. 访谈结束后，AI **只抛 1 个最强主线候选**（不是 3 个），用 50 字内复述给用户：

```
听完你这周的描述，我想把这期的主线定为：

「[主线冲突复述，50 字内]」

理由：[一句话说明为什么这个主线最有张力]

你看这个方向对吗？或者你想改成什么？
```

2. **等用户回复**：用户确认 / 改写 / 重新挖
3. 用户确认后，把主线标题作为文件夹名的一部分：

```bash
# 主线标题简化（去空格、去特殊字符，保留中文）
SLUG="删了3篇10w+的稿子"  # 用户确认后的主线短句
FOLDER_NAME="W${WEEK_NUM}-${SLUG}"
FOLDER_PATH="weekly-reports/${YEAR}/${FOLDER_NAME}"

mkdir -p "${FOLDER_PATH}/05-发布包"
# 创建工作文件（访谈记录已经在内存中，直接写入）
```

5. 把访谈记录、主线冲突、本期编号一起写入 `01-访谈记录.md`

### 步骤 3：初稿生成 + 5 轮迭代优化

**3.0 生成初稿**：

读取 `references/prompt-draft-weekly.md` 中的初稿生成 Prompt：
- 严格按 `references/template-weekly-structure.md` 的结构生成
- 内容 100% 来自访谈记录，禁止凭空生成事件
- 风格 100% 贴合主风格报告
- 主线冲突必须出现在标题 + 开篇 100 字内
- 写入 `02-初稿.md`

**3.1 - 3.5 五轮优化（魔改 draft-optimizer）**：

读取 `references/prompt-rounds-1-5-weekly.md`，按以下顺序执行：

| 轮次 | 优化维度 | 周报特有关切点 |
|------|---------|--------------|
| 第 1 轮 | 主线贯穿度 | 每个段落是否都呼应主线冲突？删除游离的段落 |
| 第 2 轮 | 故事张力 | 开场 100 字是否抓人？转折是否有画面感？事件是否具体？ |
| 第 3 轮 | 语言风格 | 贴合主风格报告；周报允许更口语、更"不确定"的表达 |
| 第 4 轮 | 追更感注入 | 数据仪表盘清晰、下周预告留悬念、本周一问能引互动 |
| 第 5 轮 | 定型 + 清 AI 味 | 清除"不是A而是B"、工整三段排比等 AI 痕迹 |

每轮覆盖 `02-初稿.md`，追加 `03-改稿记录.md`，第 4 轮起写入 `04-定稿.md`。

### 步骤 4：AI 味扫描（推荐）

调用 `/dbs-ai-check` 对 `04-定稿.md` 做扫描，仅扫描不修改。如有严重 AI 味特征，回到第 3 轮修复。

### 步骤 5：注入追更元素（周报独有）

读取 `references/recurring-elements.md`，在 `04-定稿.md` 末尾或对应位置注入：

1. **标题加编号**：`智昊周报 #012｜这周我删了3篇10万+的稿子`
2. **本周一图**：留 `![[占位]]` 让用户后续补图（或调用 `/cover-generator`）
3. **数据仪表盘**（自动填充可计算字段）：
   - 💰 收入/变现：[访谈第 3 题回答]
   - 📚 本周学到的一句话：[访谈第 6 题"意外"中提取]
   - 📊 内容产出：完成 X 篇 / 发布 X 篇（来自 Step 1 扫描结果）
4. **下周预告**：从访谈「下周悬念」阶段问答改写为悬念句，控制在 30 字内
5. **本周一问**：基于本周主线生成 1 个开放性问题，引导读者评论

### 步骤 6：生成多平台发布包

调用 `/content-publisher` 生成 4 个版本到 `05-发布包/`：

| 文件名 | 适配平台 | 长度 |
|--------|---------|------|
| `长文版.md` | 微信公众号、X Articles、Substack | 完整版，含追更元素 |
| `图文版.md` | 即刻、知识星球 | 800 字内，删数据仪表盘，强化主线故事 |
| `短版.md` | Twitter thread / X 长文摘要 | 5-8 条推文，主线开场，每条一个事件 |
| `邮件版.md` | Substack / 邮件订阅 | 加邮件专用开头「读者朋友们，本周好」 |

如果用户需要发布：
- 公众号/即刻/小红书等 → 建议运行 `/wechatsync`
- X Articles → 建议运行 `/x-article-publisher`

### 步骤 7：发布后复盘（可选·推荐）

发布后 48 小时运行 `/dbs-content` 做内容五维诊断 + `/style-updater` 让周报风格进入飞轮。

## 注意事项

- **主线冲突是周报生死线**：没有主线的周报就是流水账。第 2 步宁可多花 2 分钟，也要挖出一个有张力的主线
- **追更机制要克制**：固定栏目不要超过 4 个，否则读者会审美疲劳
- **数据仪表盘要诚实**：粉丝数据可以从缺，但收入/学习/产出每周都要有（哪怕写 0）
- **下周预告必须能兑现**：预告里说的事，下一期必须真的写了。连续 2 期不兑现会摧毁读者信任
- **访谈节奏要轻**：周报访谈不是深度访谈，单题回答别超过 200 字，全程控制在 15 分钟内

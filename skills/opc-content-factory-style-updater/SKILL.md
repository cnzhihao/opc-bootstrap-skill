# 更新写作风格分析报告 + 验证风格定型
## 脚本化执行

当任务涉及本 Skill 的飞书 CLI 操作、数据处理或批量校验时，先读取 `references/script-prompts.md`，按其中定义的脚本清单、参数、确认门禁和验收标准执行。

脚本应归属于本 Skill 包内的 `scripts/` 目录；飞书 CLI 操作默认使用 `.mjs` 并通过 `$BOX_AGENT_NODE` 执行，数据处理脚本可使用 `.py` 并通过 `$BOX_AGENT_PYTHON` 执行。执行真实脚本前先确认脚本文件存在。

涉及飞书写入、归档、发布、删除、状态迁移或对外承诺时，先向用户展示拟操作清单，获得明确确认后再执行；写入型脚本默认 dry-run，确认后才使用 execute 模式。

对比作者修订定稿与现有风格报告，双向更新报告，验证风格一致性。

## 前置条件

- 作者已审阅并修订了 `04-定稿.md`
- 写作风格分析报告已存在（`00-模板库/作者个人写作风格分析报告.md`）

## 两阶段工作流

### 阶段一：更新写作风格分析报告

读取 `references/prompt-update-style.md` 中的完整 Prompt 执行：

1. **正向对比（发现新增内容）**
   - 逐维度对比修订稿和现有报告
   - 发现作者修订中新增的表达方式、用词习惯、语气特点
   - 发现作者删掉的内容（AI味，加入"绝对不能做"清单）
   - 发现作者调整过的金句、结尾方式等

2. **反向对比（发现应删减的内容）**
   - 遍历报告中的每个公式、模式、金句、口语表达
   - 检查哪些特征在近期文章中未再出现
   - 标记为"待观察"或"建议移除"

3. **输出差异清单**（三部分）：
   - 📌 新增内容
   - ✏️ 修正内容
   - 🗑️ 建议删减（标注连续未出现篇数）

4. **等待用户确认**后，将更新写入风格分析报告

### 阶段二：验证风格定型效果

读取 `references/prompt-verify-style.md` 中的完整 Prompt 执行：

1. 对比历史文章和风格分析报告，分析定稿的风格一致性
2. 指出可能还有的 AI 痕迹
3. 给出明确的「风格是否定型」判断：
   - **已定型**：说明成功的关键因素
   - **未定型**：说明具体原因和改进建议

## 完成后

- 告知用户风格报告已更新
- 告知风格验证结果
- 如已定型，建议使用 `/content-publisher` 生成多平台发布包

## 注意事项

- 保持报告原有结构和格式，只更新具体内容
- 不直接删除内容，用 HTML 注释标注待删减
- 如果某维度无需更新，说明"与现有报告一致，无需调整"

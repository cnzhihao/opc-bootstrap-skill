# 深度访谈全流程

从选题进入访谈，创建稿件工作空间，逐题深度访谈，归档记录，确定标题。

## 前置条件

- 已从 `01-选题库/` 中选定一个高优先级选题
- 用户需要提供：@引用选定的选题笔记 + 文章类型（干货教程/经验分享/观点文/种草文/故事文）

## 四阶段工作流

### 阶段一：创建稿件工作空间（自动）

1. 从选题笔记中提取「选题标题」
2. 检查稿件文件夹是否已存在，存在则跳到步骤 5
3. 执行 bash 脚本创建文件夹结构：

```bash
TITLE="实际的稿件标题"
DATE_SUFFIX=$(date +%Y%m%d)
FOLDER_NAME="${TITLE}-${DATE_SUFFIX}"
FOLDER_PATH="02-稿件库/01-创作中稿件/${FOLDER_NAME}"

mkdir -p "${FOLDER_PATH}/assets"

CURRENT_DATE=$(date +%Y-%m-%d)
sed "s/{{NAME}}/${TITLE}/g; s/{{DATE:YYYY-MM-DD}}/${CURRENT_DATE}/g" "00-模板库/模板-访谈记录.md" > "${FOLDER_PATH}/01-访谈记录.md"

printf "# %s - 初稿\n\n" "${TITLE}" > "${FOLDER_PATH}/02-初稿.md"
printf "# %s - 改稿记录\n\n" "${TITLE}" > "${FOLDER_PATH}/03-改稿记录.md"
printf "# %s - 定稿\n\n" "${TITLE}" > "${FOLDER_PATH}/04-定稿.md"

echo "稿件工作空间创建完成：${FOLDER_PATH}"
```

4. 读取生成的 `01-访谈记录.md` 确认模板填充正确
5. 向用户确认文件夹路径，进入阶段二

### 阶段二：深度访谈（人工+自动协作）

读取 `references/prompt-interview.md` 中的完整访谈 Prompt 执行。核心要点：

- **每次只问 1 个问题**，等待用户回答后再问下一个
- **动态追问**，根据回答调整问题方向
- **四层推进路径**：背景引入 → 痛点共鸣 → 干货实操 → 认知升华
- **追问机制**：回答过短、有趣经历未展开、偏通用、关键信息一笔带过时追问
- **结束判断**：素材覆盖完整内容链路时可结束

### 阶段三：归档访谈记录（自动）

1. 将完整问答写入稿件文件夹的 `01-访谈记录.md`
2. 补充「访谈核心观点提炼」：3-5 个核心观点、金句、骨架建议、差异化分析
3. 向用户确认关键发现和亮点

### 阶段四：标题选择（人工确认）

1. 生成 5 个备选标题
2. **标题必须严格符合个人风格**：读取 `00-模板库/作者个人写作风格分析报告.md` 中的「标题风格」维度，遵守以下规则：
   - 口语化、有态度、有第一人称、有具体数字/事件
   - 标题本身就是一个微型故事，读完就想点进去
   - 禁止论文腔/散文腔/标语腔/鸡汤腔
   - 禁止"不是A，而是B"的AI味对比句式
   - 禁止抽象概念堆砌、数学符号（≠≥≤）、悬而未决问句
   - 优先使用风格报告中的5个标题公式
3. 等待用户选择或修改
4. 将确定标题追加到 `01-访谈记录.md`
5. 建议用户下一步使用 `/draft-optimizer` 生成初稿并优化

## 注意事项

- 问题必须围绕个人经历、实操经验、独家观点，禁止通用问题
- 记录用户回答原文，不要总结或改写
- 访谈节奏要自然，像好奇的朋友，不要像审讯

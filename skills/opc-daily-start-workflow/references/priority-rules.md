# OPC 每日优先级规则

从高到低判断：

1. 现金流/回款：已交付待回款、今天要确认的钱、社保公积金压力。
2. 客户成交：报价、跟进、需求确认、成交推进。
3. 已承诺交付：已经收钱或明确承诺的交付。
4. 本周主战役：本周 1-2 个最重要突破点。
5. 内容获客：服务于收入或渠道的内容。
6. 长期资产：蓝皮书、SOP、产品化资料、案例。
7. 学习探索：除非直接服务于主战役，否则不进入今日必须项。

如果用户当日可投入时间少于 2 小时，只保留 1 个现金流/客户动作和 1 个最小收口动作。

## 合并规则

优先级判断完成后，不要把所有事项逐条输出。必须先合并成今日主线：

- 今日主线最多 1 条；
- 必须动作最多 3 个；
- 不能服务今日主线的事项进入暂停/不做池；
- 新想法默认不进入今日主线，除非能直接带来现金流、客户信任、产品验证或关键确定性。

## 48 小时突破规则

每日启动必须给出一个 48 小时突破点。突破点优先从以下事项中选择：

1. 最接近钱的动作；
2. 最能恢复或增强客户信任的动作；
3. 最能验证产品/项目是否值得继续推的动作；
4. 最能解除本周主战役卡点的动作。

突破点必须包含第一动作和完成标准。

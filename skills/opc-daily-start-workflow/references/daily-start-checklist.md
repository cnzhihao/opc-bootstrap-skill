# OPC 每日启动检查清单

每日启动不是列 todo，而是把今天的注意力压缩到一条主线、三个必须动作和一个 48 小时突破点。

## 1. 梳理：今天真实局面是什么

先快速读取或询问以下信息：

- 现金流：今天/本周有没有待回款、待报价、待成交、待催收事项；
- 客户：有没有已经承诺、已经交付、正在卡住、需要维护信任的客户事项；
- 项目：有没有本周必须推进的产品、demo、交付、合作；
- 内容：有没有能服务获客、成交、产品验证的内容动作；
- 系统：有没有必须同步到飞书经营系统的进展；
- 噪音：今天出现了哪些想做但不服务主战役的诱惑。

梳理阶段只分类，不直接安排日程。

## 2. 合并：今天只保留一条主线

把所有事项合并成：

- 今日主线：最多 1 条；
- 必须动作：最多 3 个；
- 暂停/不做池：不限；
- 需要等待的信息：只列会影响今日主线的部分。

合并时按以下顺序判断：

1. 会不会影响现金流、回款、成交；
2. 会不会影响已有客户信任；
3. 会不会推进本周主战役；
4. 能不能复用成产品、内容、案例或方法论资产；
5. 今天不做会不会造成不可逆损失。

不满足以上条件的事项，默认进入暂停/不做池。

## 3. 突破：确定 48 小时突破点

每日启动必须从今日主线中选出一个 48 小时突破点。

突破点必须回答：

- 突破点是什么；
- 为什么今天先做它；
- 第一动作是什么；
- 完成标准是什么；
- 今天明确不做什么。

## 5. 反模式

不要输出十几条待办。
不要平均安排上午下午晚上。
不要把新想法直接塞进今日计划。
不要没有暂停池。
不要没有 48 小时突破点。
不要只说“建议优先做 A”，必须给出第一动作和完成标准。
